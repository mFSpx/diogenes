# Board Effect Tournament Law

Status: ACTIVE

Santa and Krampus are LUCIDOTA-local routing metaphors, not external technical terms.

## Core law

Every move affects the board.

A move must be scored by the number of board systems it improves or hardens:
- proof / receipts
- graph / staged candidates
- routing / gates
- model fabric / delegation
- corpus / Krampuschewing / Indy_READs
- tests / verification
- ontology / working reality
- status / handoff / observation
- database / SQL / queues
- latency / async execution

## Krampus

Krampus is the fair audit/enforcement persona.

Krampus prefers moves that affect `2` or `4+` board systems.
Krampus may classify naughty/slop only from evidence.
Krampus must not imagine wrongdoing, slop, or corruption without receipts, file paths, DB rows, logs, tests, or contradiction records.

Krampus wins by reducing entropy without destroying proof.

## Santa

Santa is the glow-finding/exploration persona.

Santa may settle for one board effect.
Santa may assume and search for new glow, but assumptions must be labeled as assumptions and converted into testable working realities before authority.
Santa is allowed to be optimistic; Santa is not allowed to become canonical truth without receipts.

## Tournament mode

LUCIDOTA is a solo project-builder game where the system drafts the game, plays it with the operator, plays bounded subgames against itself, and recruits models/algorithms/workflows as pieces.

The season score is:
- more truth under receipt discipline
- lower operator friction
- higher leverage per move
- lower unnecessary visibility/noise
- higher glow from verified discoveries
- stronger graph and queue position
- better comparison from February hypertimeline to now

Explore/exploit means: search weak signals, test hypotheses, automate bounded workflows, and eliminate slop paths. It does not mean direct canonical graph mutation, destructive deletion, or unevidenced accusation.
